现场环境配置

## 一、部署目录结构

项目部署在服务器 `/data/app/analysis-platform` 目录下，结构如下：

    analysis-platform           # 项目根部署目录
    ├── config                  # 项目核心配置文件目录
    ├── data                    # 货场静态数据文件总目录
    │   ├── shp                 # 货场SHP地理信息文件存放目录
    │   └── tiles               # 货场影像瓦片文件存放目录
    ├── dist                    # 前端静态资源部署目录
    ├── logs                    # 项目运行日志文件目录
    ├── analysis-platform.jar   # 项目后端启动Jar包
    ├── analysis-platform.pid   # 项目进程PID文件
    └── restart.sh              # 项目重启脚本

## 二、部署说明
  
   ### 1. 货场 SHP 文件部署
    - 存放路径：`/data/app/analysis-platform/data/shp/`
    - 配置对应规则：
     ```
     需要按照对应货场的 yml 文件中experiment.file-path.yards.${yard-name}的配置，将货场 SHP 文件上传到data/shp目录中。
     ```
  
   ### 2. 货场影像瓦片部署
  
    - 存放路径：`/data/app/analysis-platform/data/tiles/`
    - 配置对应规则
    ```
    需要按照对应货场的要求，将影像瓦片文件上传到data/tiles目录中。
     ```

## 三、YML 配置修改
  
   #### 3.1 全局基础配置
   - `experiment.platform-dir`
     ```
     需要改为货场应用地址：/data/app/analysis-platform
     ```
     
   #### 3.2 货场基础配置
   - `experiment.base.push-data`
     ```
     需要设置为 true
     ```
   - `experiment.base.yard-name`
     ```
     需要改为当前部署的货场名称
     ```
   - 其他配置项可按需调整
  
   #### 3.3 真趣平台对接配置
   - `experiment.base.joysuch.building-name`
     ```
     填写货场名称（汉字）
     ```
   - `experiment.base.joysuch.building-id`
     ```
     填写真趣平台分配的建筑 ID
     ```

   #### 3.4 卡片行为识别配置
   - `experiment.base.card-analysis.visual-identify.base-url`
     ```
     视觉识别服务地址，按需调整
     ```
   - `experiment.base.card-analysis.visual-identify.camera-ids`
     ```
     对应「定位卡行为识别」的摄像头 ID
     ```
   - `experiment.base.card-analysis.visual-identify.event-types`
     ```
     对应「定位卡行为识别」的事件类型
     ```
   - `experiment.base.card-analysis.visual-identify.unmatch-camera-ids`
     ```
     对应「无定位卡行为识别」的摄像头 ID
     ```
   - `experiment.base.card-analysis.visual-identify.unmatch-event-types`
     ```
     对应「无定位卡行为识别」的事件类型
     ```
  
   ### 四、Nginx 配置部署
   - 配置文件存放位置：
     ```
     项目目录下的config目录
     ```
   - 部署操作：
     ```
     将config目录下的 Nginx 配置文件，放置到服务器 Nginx 的配置目录中，重载 Nginx 配置使其生效。
     ```
