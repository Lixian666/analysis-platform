现场环境配置

部署目录结构

├── /data/app/analysis-platform
│   └── config                  # 项目配置文件目录
│   └── data                    # 货场数据文件目录
│       └── shp                 # 货场shp文件目录
│       └── tiles               # 货场影像瓦片目录
│   └── dist                    # 前端文件目录
│   └── logs                    # 日志文件目录
│   └── config                  # 项目配置文件目录
├── analysis-platform.jar       
├── analysis-platform.pid
├── restart.sh

部署说明

1、data目录的shp目录中存放货场shp文件，需要按照对应货场的yml文件中（experiment.file-path.yards.${yard-name}）的配置将shp文件上传到data目录的shp目录中
2、data目录的tiles目录中存放货场影像瓦片，需要按照对应货场的将影像瓦片上传到data目录的tiles目录中
3、部署配置
    yml配置：
        experiment
            platform-dir需要改为货场应用地址（"/data/app/analysis-platform"）
        experiment.base
            push-data、push-data需要设置为true
            yard-name需要改为货场的名称
            其他配置按需调整
        experiment.base.joysuch
            building-name: 货场名称汉字
            building-id: 真趣平台建筑id
        experiment.base.card-analysis
            visual-identify: base-url、camera-ids、event-types、unmatch-camera-ids、unmatch-event-types按需调整
            camera-ids、event-types对应定位卡进行行为识别的功能
            unmatch-camera-ids、unmatch-event-types对应无定位卡进行行为识别的功能
4、nginx配置文件放置到nginx配置目录中，目录[config](config)
